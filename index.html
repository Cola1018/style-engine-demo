    How it's actually built

    Nothing here is hand-tuned per style. The same four steps run for every site — which is the whole point. The first site pays for the pipeline; the rest ride it.

    Step one

    Capture the brief

    Client copy, sections and proof points go into content.json. Written once, never touched again.

    Step two

    Read the reference

    A reference site or screenshot goes in; a tokens.json comes out — type, scale, colour, border, shadow, rhythm.

    Step three

    Compose

    A reusable Claude Code skill binds content to tokens against one component set. No bespoke markup per client.

    Step four

    Review & ship

    Contrast, focus states and mobile pass are checked before handoff. A human signs off on every site.
